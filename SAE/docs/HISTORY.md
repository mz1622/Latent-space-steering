Added COCO JSONL loading, top-k token pooling, image-level pooled SAE sample construction, mention-level sample expansion, and direct mean-difference helpers for object presence and supported-vs-hallucinated alignment.
- Strengthened the default sparsity behavior in [`SAE.py`](/home/ec2-user/Latent-space-steering/SAE/src/method/SAE.py) and [`train_SAE.py`](/home/ec2-user/Latent-space-steering/SAE/scripts/train_SAE.py). The SAE now always trains in `torch.float32`, uses an explicit ReLU-latent sparsity loss `latent.mean() * l1_coeff`, logs density / active latents per sample / mean latent / dead-feature fraction each epoch, and raises the default `l1_coeff` from `1e-4` to `1e-3`.
- Hardened [`SAE.py`](/home/ec2-user/Latent-space-steering/SAE/src/method/SAE.py) to match the local `liuhaotian/llava-v1.5` structure more precisely. The extractor now resolves the LLaVA vision tower wrapper explicitly, forwards through the real CLIP vision model with `output_hidden_states=True`, treats inputs as preprocessed `pixel_values_bchw`, handles CLS tokens conditionally based on sequence length vs. `num_patches`, and keeps `flat_nd` consistently shaped `[N, d]`.
- Fixed NaN SAE training in [`SAE.py`](/home/ec2-user/Latent-space-steering/SAE/src/method/SAE.py) and [`train_SAE.py`](/home/ec2-user/Latent-space-steering/SAE/scripts/train_SAE.py). The bug was that the SAE was being trained in the LLaVA model dtype, which is typically `float16`. The SAE training path now casts activations to `float32`, constructs the SAE in `float32`, and asserts on non-finite activations, losses, gradients, and parameters so failures stop immediately instead of silently saving broken checkpoints.
- Added [`data_loader.py`](/home/ec2-user/Latent-space-steering/SAE/src/data_loader/data_loader.py) with a minimal image-only MSCOCO dataset/dataloader that reads from `SAE/data/MSCOCO/train2014` and preprocesses images with the LLaVA image processor.
- Added [`__init__.py`](/home/ec2-user/Latent-space-steering/SAE/src/data_loader/__init__.py) to expose the dataloader entry points.
- Added [`train_SAE.py`](/home/ec2-user/Latent-space-steering/SAE/scripts/train_SAE.py) to load LLaVA, iterate over MSCOCO images, extract selected vision-layer activations, train one SAE per layer, and save checkpoints under `SAE/outputs/sae/`.
- Added a minimal [`README.md`](/home/ec2-user/Latent-space-steering/SAE/README.md) describing the current SAE workstream and the main training command.
